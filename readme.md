# HardNet++: Nonlinear Constraint Enforcement in Neural Networks

Enforcing constraint satisfaction in neural network outputs is critical for safety, reliability, and physical fidelity in many control and decision-making applications. Many real-world problems of interest are nonlinear and do not admit a tractable projection onto the feasible set, motivating the development of methods that can enforce general nonlinear constraints. To this end, we introduce **HardNet++**, a constraint-satisfaction method that enforces linear and nonlinear equality and inequality constraints. Our approach iteratively adjusts the network output via damped local linearizations of the constraints. Each iteration is differentiable, admitting an end-to-end training framework, where the constraint satisfaction layer is active during training.

## Structure
- `mpc_obst.py`: trajectory generation for training
- `mpc_dataset.npz`: trajectories generated by `mpc_obst.py`
- `model.py`: **HardNet++** model
- `model_dc3.py`: DC3 model for benchmarking
- `train.py`: training for **HardNet++** model and benchmark models
- `eval.py`: computes average and maximum constraint violation and suboptimality for **HardNet++** model and benchmark models
- `paper_figs/`: contains the best-performing trained models for **HardNet++** and the DC3 and soft-constrained benchmarks 
  - `paper_figs.ipynb` plots a sample closed-loop trajectory for **HardNet++**, DC3, and soft-constrained models along with a solver baseline

## Dependencies
This code requires Python with `torch`, `numpy`, `scipy`, `matplotlib`, `tqdm`, and `casadi`.

## Usage
To train the **HardNet++** model used in our paper  "[HardNet++: Nonlinear Constraint Enforcement in Neural Networks](https://arxiv.org/pdf/2604.19669)":
```bash
python train.py --constrained
```
This uses the best-performing HardNet++ hyperparameters from our experiments: `lr`=1e-4, `epsilon`=0.3, `n_proj`=500, `lambda`=0.0 by default. 

To train the DC3 benchmark model used in our paper:
```bash
python train.py --dc3 --lambda 10.0 --lr 1e-5
```
This uses the best-performing DC3 hyperparameters from our experiments: `lr`=1e-5, `lambda`=10.0, `dc3_steps`=100, `dc3_stepsize`=1e-1  

To train the soft-constrained benchmark model used in our paper:
```bash
python train.py --lambda 1000.0
```
This uses the best-performing soft-constrained hyperparameters from our experiments:  `lr`=1e-4, `lambda`=1000.0

See train.py for relevant flags and their usage. To train multiple models with different hyperparameters, one can run a bash script, with, for example:
```bash
for eps in 0.1 1 10; do
    python train.py --constrained --epsilon $eps
done
```

To evaluate the trained models on unseen initial conditions and compute average and maximum suboptimality and constraint violation as reported in the paper: 
```bash
python eval.py /path/to/model/folder/
```
